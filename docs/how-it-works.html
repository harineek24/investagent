How InvestAgent Works

A lecture-style walkthrough of every algorithm, agent, and design decision inside a multi-agent AI hedge fund simulator.

1. System Overview

InvestAgent is a multi-agent system built on LangGraph. Instead of a single prompt producing an answer, six autonomous analyst agents each independently research a stock, debate when they disagree, and a portfolio manager makes the final call. Everything runs on free-tier LLMs.

User Input (tickers, date range, cash) | v +-------------------------------------------------------+ | 6 Analyst Agents run in PARALLEL | | (Value, Growth, Contrarian, Technical, | | Fundamental, Sentiment) | | Each runs a ReAct loop: Think -> Act -> Observe x4 | +-------------------------------------------------------+ | v Agreement Check ── "Do the agents agree?" | (measure consensus) | +-- >= 60% agree -----> Risk Manager | +-- < 60% agree ------> Debate (LLM moderator) | v Risk Manager | v Portfolio Manager (multi-turn ReAct agent) | v Final BUY / SELL / HOLD + Save to Memory (SQLite)
What makes this "agentic" vs. a regular LLM call? Each agent autonomously decides which tools to call and how many times to loop. The debate only happens when there's real disagreement. The portfolio manager can fetch additional data before deciding. None of this is hardcoded — the LLM controls the flow.

2. The ReAct Agent Pattern

ReAct (Reasoning + Acting) is an agent design pattern from the 2022 paper by Yao et al. The idea is simple: instead of the LLM answering in one shot, it enters a loop:

  1. Think — "I need the financial metrics to check the P/E ratio."
  2. Act — Calls get_financial_metrics({"ticker": "AAPL"})
  3. Observe — Reads the result: P/E = 28.5, ROE = 147%, ...
  4. Repeat — "Now I need price data for technical confirmation."
  5. Conclude — Emits a final signal: bullish / neutral / bearish + confidence

Each agent gets up to 4 tool calls per ticker (the MAX_ITERATIONS constant). This soft limit prevents runaway API costs on free tiers while still allowing meaningful research.

How tool calls work under the hood

The agent outputs a special format that the system parses:

TOOL_CALL: get_financial_metrics({"ticker": "AAPL"})

The runner extracts the function name and JSON arguments, executes the tool, and appends the result back into the conversation. Key parameters like ticker, start_date, and end_date are auto-injected so the agent doesn't waste tokens repeating them.

If the loop exhausts without concluding

If an agent uses all 4 iterations without emitting a FINAL_ANSWER, the system returns a neutral signal with 30% confidence — a safe fallback that effectively says "I couldn't form a strong opinion."

// Fallback when ReAct loop exhausts
{
    "signal": "neutral",
    "confidence": 30,
    "reasoning": "Unable to form conviction within iteration limit."
}

3. The Six Analyst Agents

Each analyst has a distinct investment philosophy embedded in its system prompt and a scoring algorithm that converts financial data into a signal. They all share the same 7 tools, but each cares about different metrics.

3.1 Value Analyst

Philosophy: Buffett & Graham — buy quality companies trading below intrinsic value. Looks for a "margin of safety" between price and what the business is actually worth.

MetricBullishNeutralBearish
ROE>15% (+2)10-15% (+1)<5% (-1)
Profit Margin>20% (+2)10-20% (+1)<0% (-1)
Debt/Equity<50 (+2)50-100 (+1)>100 (-1)
Current Ratio>1.5 (+1)1.0-1.5 (0)<1.0 (-1)
P/E Ratio0-15 (+2)15-20 (+1)>30 (-1)
P/B Ratio0-1.5 (+2)1.5-3 (+1)>5 (-1)
FCF Yield>8% (+2)5-8% (+1)—

Signal logic: Score ranges from -12 to +12. Score ≥ 4 = bullish, score ≤ -2 = bearish, otherwise neutral. Confidence = min(|score| / max_score, 1.0) × 100.

3.2 Growth Analyst

Philosophy: Peter Lynch & Cathie Wood — find companies growing revenue and earnings faster than the market. Willing to pay premium valuations for high growth.

MetricBullishBearish
Revenue Growth>25% (+3)<0% (-2)
Earnings Growth>25% (+3)<0% (-2)
PEG Ratio<1.0 (+2)>3.0 (-1)
Operating Margin>25% (+2)—
20-Day Return>10% (+1)<-15% (-1)
Why PEG matters The PEG ratio (P/E divided by earnings growth rate) normalizes valuation for growth. A PEG < 1.0 means you're paying less than "1x" for each point of growth — Lynch's favorite signal that a stock is undervalued relative to its trajectory.

3.3 Contrarian Analyst

Philosophy: Michael Burry & Bill Ackman — "buy fear, sell greed." Looks for stocks the market has beaten down too far, as long as the business is fundamentally sound.

SignalConditionPoints
Near 52-week lowPrice within bottom 30%+2
Near 52-week highPrice within top 10%-2
Discount to 200D MA>20% below average+2
Premium to 200D MA>20% above average-1
Solid fundamentalsPositive FCF + decent metrics+2
High volatility>40%+1
Low P/E<12+2

The key insight: contrarian analysis inverts the signals that most analysts fear. Where the sentiment agent sees danger, the contrarian sees opportunity.

3.4 Technical Analyst

Philosophy: Pure price action. Doesn't care about the company's business — only what the chart says about momentum, trend, and mean reversion.

Indicator Suite

1. Exponential Moving Averages (EMA) — Three periods: 8 (fast), 21 (medium), 55 (slow).

EMA8 > EMA21 > EMA55  →  Strong uptrend  (+2)
EMA8 > EMA21          →  Uptrend         (+1)
EMA8 < EMA21 < EMA55  →  Strong downtrend (-2)
EMA8 < EMA21          →  Downtrend       (-1)

2. RSI (Relative Strength Index, period = 14) — Measures momentum on a 0-100 scale.

RSI < 30   →  Oversold   (+2)   ← "too much selling, bounce expected"
RSI > 70   →  Overbought (-2)   ← "too much buying, pullback expected"

3. MACD (Moving Average Convergence Divergence) — Trend-following momentum indicator.

Histogram crosses above 0  →  Bullish crossover (+2)
Histogram > 0              →  Bullish           (+1)
Histogram crosses below 0  →  Bearish crossover (-2)
Histogram < 0              →  Bearish           (-1)

4. Bollinger Bands (20-period, 2 standard deviations) — Mean-reversion signal.

Price below lower band  →  Oversold  (+1)
Price above upper band  →  Overbought (-1)

5. Volume Confirmation — If volume is >1.5x the 20-day average and the score is already directional, add +1 or -1. High volume validates conviction.

Signal threshold: score ≥ 3 = bullish, score ≤ -3 = bearish.

3.5 Fundamental Analyst

Philosophy: Holistic analysis across four dimensions. Unlike the value agent (which focuses on cheapness) or growth (which focuses on trajectory), the fundamental analyst tries to see the complete picture.

Each dimension scores independently from -2 to +2, then they're summed:

DimensionMetricsRange
ProfitabilityROE, Profit Margin, Operating Margin-3 to +5
Financial HealthDebt/Equity, Current Ratio, FCF-3 to +4
ValuationP/E, P/B, P/S-4 to +4
GrowthRevenue Growth, Earnings Growth-2 to +4

Total range is roughly -8 to +8. Score ≥ 5 = bullish, ≤ -3 = bearish.

3.6 Sentiment Analyst

Philosophy: The market is driven by people. Track what they're saying, what insiders are doing, and what Wall Street recommends.

Three independent signal sources:

  1. News Sentiment — Keyword matching on recent headlines. Counts positive words (beat, surge, upgrade, record) vs. negative (miss, crash, downgrade, weak). Net positive > 2 = +2, net negative < -2 = -2.
  2. Insider Trades — If company executives are buying their own stock, that's a vote of confidence (+1). If they're selling more than 2x buys, that's a warning (-1).
  3. Analyst Recommendations — Buy upgrades vs. sell downgrades from Wall Street analysts. More buys = +1, more sells = -1.

Combined score range: -4 to +4. Score ≥ 2 = bullish, ≤ -2 = bearish.

4. Agreement Check & Debate

After all six analysts finish, the system asks: "Do they agree?"

Measuring consensus

For each ticker, count the signal distribution:

AAPL signals: [bullish, bullish, bullish, bearish, bearish, neutral]
Majority: bullish (3 out of 6) = 50% agreement

MSFT signals: [bullish, bullish, bullish, bullish, neutral, neutral]
Majority: bullish (4 out of 6) = 67% agreement

Overall agreement = average(50%, 67%) = 58.5%

If overall agreement < 60% threshold, the Debate node activates.

The Debate mechanism

A senior investment committee moderator (an LLM with a special prompt) receives:

  1. Every analyst's signal, confidence, and reasoning
  2. Each analyst's historical track record from the memory database

The moderator's job is to produce a synthesis:

{
    "synthesis": "Value and Fundamental see strong margins, but Technical
                  shows a bearish EMA crossover. The bearish technicals
                  are short-term; fundamentals dominate on this timeframe.",
    "leaning": "bullish",
    "confidence_adjustment": +10,
    "key_disagreement": "Short-term momentum vs. long-term value",
    "strongest_argument": "Value Analyst (75% historical accuracy) —
                          P/E of 14 with 22% ROE is a clear margin of safety"
}
Why track records matter in debate If the Value Analyst has 75% historical accuracy over 48 signals and the Technical Analyst has 58% over 30 signals, the moderator can rationally weight the value argument higher. This is a form of meta-learning — the system learns which of its own agents to trust more over time.

5. Risk Manager

Before the Portfolio Manager can act, the Risk Manager enforces hard portfolio constraints. This prevents any single stock from dominating the portfolio.

The 20% Rule

Portfolio total value = cash + sum(position values)
Max per ticker        = 20% of total
Remaining limit       = max(0, max_per_ticker - current_position_value)
Final limit           = min(remaining_limit, available_cash)
Max shares            = floor(final_limit / current_price)

Example:

Portfolio:  $100,000 cash + $50,000 positions = $150,000 total
20% max:    $30,000 per ticker
AAPL held:  $10,000 currently
Remaining:  $30,000 - $10,000 = $20,000
Cash OK:    $100,000 available (no constraint)
Final:      Can buy up to $20,000 more of AAPL

The Portfolio Manager receives these limits and is never allowed to exceed them, no matter how bullish the analysts are. Quantities are clamped after the PM decides.

6. Portfolio Manager

The PM is the final decision maker and is itself a ReAct agent with up to 3 tool calls per ticker. It has the most context of any node in the system.

Information available to the PM

InputSource
6 analyst signals + confidenceAnalyst nodes
Debate synthesis (if it happened)Debate node
Each analyst's historical accuracyMemory DB
Past decisions on this tickerMemory DB
Risk limits (max shares, remaining $)Risk Manager
Current portfolio (cash, positions)State
Additional tool accessAPI tools

Decision rules embedded in the prompt

  1. Weight analyst signals by their track record accuracy
  2. If debate occurred, seriously consider the synthesis
  3. Only buy with strong conviction AND available limit
  4. Only sell if holding shares AND bearish conviction
  5. Never exceed the risk manager's max shares
  6. Be conservative — when in doubt, hold
  7. Explain the reasoning chain, not just the conclusion

Quantity clamping

Even after the PM decides, a safety check ensures constraints are respected:

if action == "buy":
    quantity = min(pm_quantity, risk_manager_max_shares)
elif action == "sell":
    quantity = min(pm_quantity, shares_currently_held)

7. Memory & Learning

InvestAgent uses a SQLite database (investagent_memory.db) to persist decisions across sessions. This is what separates it from a stateless LLM wrapper.

Three tables

TableStoresUsed by
decisions Every agent signal, PM action, price at decision, 30-day outcome PM (reviews past decisions on this ticker)
runs Run metadata: tickers, provider, return % Dashboard, backtester
agent_scores Per-agent accuracy: signal, was_correct (1/0) Debate moderator, PM

The learning loop

Run 1: Agent makes prediction → saved to memory | v (30+ days later) Run N: Check actual price movement → Mark outcome as "correct" or "incorrect" → Update agent_scores | v Run N+1: Debate moderator and PM receive track records → Weight signals by historical accuracy → Better-performing agents get more influence

Over many runs, agents that are consistently wrong get discounted, and agents that are consistently right gain more influence. This is a lightweight form of online meta-learning without retraining any model weights.

8. Intelligent LLM Routing

Not all agents need the same model. The system uses a two-tier routing strategy to balance cost and capability:

Portfolio ManagersmartLlama 3.3 70BGemini 2.0 Flash
Debate ModeratorsmartLlama 3.3 70BGemini 2.0 Flash
Why this works Analyst agents do relatively simple scoring (compare metrics to thresholds). An 8B model handles this well. But the Portfolio Manager must synthesize conflicting signals, weigh track records, consider risk limits, and form a nuanced decision — that's a job for a 70B model. All of this runs on free-tier APIs.

All models use temperature = 0 (greedy decoding) for deterministic, reproducible outputs.

10. Backtesting

The backtester runs the entire multi-agent pipeline repeatedly over historical windows to simulate how the system would have performed over time.

Rolling window algorithm

|--- 90-day lookback ---|--- 30-day step ---| ^ ^ window_start window_end Step 1: Run hedge fund on window → get buy/sell/hold decisions Step 2: Execute decisions against the portfolio Step 3: Record portfolio value Step 4: Slide window forward by 30 days Step 5: Repeat until end date

Key properties:

  • Uses actual historical prices from Yahoo Finance
  • Each window's decisions compound into the next (realistic portfolio evolution)
  • 90-day lookback gives technical indicators enough history to be meaningful
  • All agent decisions are persisted to memory, enabling the learning loop

11. LangGraph Wiring

The entire workflow is expressed as a directed graph using LangGraph's StateGraph. Here's the actual wiring code, simplified:

graph = StateGraph(AgentState)

# Add all nodes
graph.add_node("Value Analyst", value_agent)
graph.add_node("Growth Analyst", growth_agent)
graph.add_node("Contrarian Analyst", contrarian_agent)
graph.add_node("Technical Analyst", technical_agent)
graph.add_node("Fundamental Analyst", fundamental_agent)
graph.add_node("Sentiment Analyst", sentiment_agent)
graph.add_node("Agreement Check", check_agreement)
graph.add_node("Debate", debate_agents)
graph.add_node("Risk Manager", risk_manager)
graph.add_node("Portfolio Manager", portfolio_manager)

# START → all analysts run in parallel
for name in analyst_names:
    graph.add_edge("__start__", name)

# All analysts → Agreement Check (waits for all to finish)
for name in analyst_names:
    graph.add_edge(name, "Agreement Check")

# Conditional branch: debate or skip
graph.add_conditional_edges(
    "Agreement Check",
    should_debate,     # returns "Debate" or "Risk Manager"
    {"Debate": "Debate", "Risk Manager": "Risk Manager"},
)

# Debate → Risk Manager → Portfolio Manager → END
graph.add_edge("Debate", "Risk Manager")
graph.add_edge("Risk Manager", "Portfolio Manager")
graph.add_edge("Portfolio Manager", END)

State management

All nodes read from and write to a shared AgentState (a typed dict). Two key patterns:

  • Message accumulation — Messages from all agents are appended to a shared list using operator.add, preserving the full conversation trail.
  • Dict merging — Each analyst adds its signals to a shared analyst_signals dict via a custom merge_dicts reducer. No agent overwrites another's output.

12. Key Takeaways

1. Agentic ≠ one big prompt Each agent autonomously decides what data to fetch. The debate only fires when there's real disagreement. The graph branches dynamically. This is what makes it "agentic."
2. Diverse perspectives beat a single model Six agents with different philosophies (value, growth, contrarian, technical, fundamental, sentiment) naturally surface different aspects of a stock. Their disagreements are information, not noise.
3. Memory enables meta-learning By tracking which agents are historically accurate and feeding that back into the debate and PM, the system learns who to trust — without retraining any weights.
4. Smart resource allocation Fast 8B models for scoring agents, capable 70B models for the PM and debate. All on free tiers. This makes the system practical to run at zero cost.
5. Risk constraints are non-negotiable The 20% per-ticker limit is enforced after the PM decides, not by the PM itself. Hard limits prevent the LLM from making catastrophic allocation errors.
Disclaimer This is an educational simulator. The agents use simplified scoring heuristics and free-tier LLMs. Real hedge funds use proprietary data, vastly more compute, and human oversight. Do not use this for actual trading decisions.
